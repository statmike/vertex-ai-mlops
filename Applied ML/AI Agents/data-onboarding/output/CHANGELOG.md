### Schema Decisions
- bats_listed_short_interest_finra: 11 columns — Daily report from FINRA on short interest positions for securities listed on the BATS exchange. It provides a snapshot of the total number of shares held short for each security for the current and previous reporting cycles.
- borrow_intensity_indicators_2023_02_01_0937: 31 columns — A snapshot of borrow intensity indicators for specific stock tickers. These indicators measure the difficulty or cost of borrowing a stock, often used as a proxy for short-selling demand. The data is provided for various rolling time windows (45, 60, 90, 180, 270, 360 days).
- cboe_equity_svr_byx: 5 columns — Daily Short Volume Report for Cboe's BYX equity exchange. This file contains the aggregated volume of shares executed as short sales for each symbol on a given trading day.
- cboe_equity_svr_bzx: 5 columns — Daily Short Volume Report for Cboe's BZX equity exchange. This file contains the aggregated volume of shares executed as short sales for each symbol on a given trading day.
- cboe_equity_svr_edga: 5 columns — Daily Short Volume Report for Cboe's EDGA equity exchange. This file contains the aggregated volume of shares executed as short sales for each symbol on a given trading day.
- cboe_equity_svr_edgx: 5 columns — Daily Short Volume Report for Cboe's EDGX equity exchange. This file contains the aggregated volume of shares executed as short sales for each symbol on a given trading day.
- cboe_equity_short_sales_edgx: 6 columns — Tick-level data for individual short sale transactions on the Cboe EDGX equity exchange. Each row represents a single short sale trade.
- cboe_australia_equity_eod_2022_05_06: 15 columns — End-of-day summary for equities traded on Cboe Australia (CXA). Provides open, high, low, close (OHLC) prices, volume, and turnover information for a given trading day.
- cboe_cfe_trades_rth: 25 columns — Trade data for Cboe Futures Exchange (CFE) products during Regular Trading Hours (RTH). Each record represents a single trade fill and includes details about the matched orders.
- ftref_early_exercise_strike_by_option_class: 6 columns — Reference data from FT Options showing the strike price thresholds for early exercise of American-style options. This is relevant for determining if an option is a candidate for early exercise, typically due to dividend payments.
- ftref_event_free_realized_volatility: 11 columns — Reference data from FT Options providing historical volatility (HV) calculations for various time periods (1 week to 2 years). The 'NetEarn' suffix suggests these volatility figures have been adjusted to exclude the impact of earnings announcements.
- ftref_fit_parameters: 23 columns — Reference data from FT Options containing fitted parameters of a volatility smile/skew model for various option expirations. These parameters (ATM Vol, Skew, Kurtosis) describe the shape of the implied volatility surface.
- ftref_implied_earnings_move: 3 columns — Reference data from FT Options showing the market's expectation for a stock's price move following its next earnings announcement. This is derived from the implied volatility of options that expire shortly after the earnings date.
- ftref_traded_vega_ratio: 7 columns — Reference data from FT Options that compares the actual traded vega (a measure of option volume weighted by sensitivity to volatility) to the average daily vega. This ratio can indicate unusual activity in volatility trading.
- high_level_option_sentiment_complete: 80 columns — A high-level daily summary of options trading sentiment for various underlying symbols. It aggregates volumes, trades, premiums, and other metrics, breaking them down by calls and puts to provide a snapshot of market sentiment.
- optsum: 17 columns — Daily end-of-day options summary data, likely from the LiveVol platform, for a specific underlying (^SPX). It provides open interest, volume, and pricing information for each option series.
- ta13_f_comp: 25 columns — Aggregated data from Form 13F filings, showing institutional investment managers' holdings in a specific symbol (IBM) as of a given quarter. It details positions in shares, puts, and calls, along with filer information.
- underlying_eodsummaries: 7 columns — End-of-day summary records for an underlying security, capturing specific trade events like the opening trade. This appears to be a granular log of trades that contribute to an EOD summary.
- underlying_eod: 12 columns — A standard end-of-day (EOD) summary for an underlying security. Provides daily open, high, low, close (OHLC) prices, volume, and bid/ask snapshots.
- underlying_intervals_900sec: 10 columns — Intraday summary data for an underlying security, aggregated into 900-second (15-minute) intervals. Each row provides OHLC, volume, VWAP, and bid/ask for one interval.
- underlying_options_eodcalcs_2023_08_25_cgi_or_historical: 34 columns — End-of-day summary for option series, including OHLC, volume, open interest, and two market snapshots (15:45 ET and EOD). This version includes calculated values like Implied Volatility and Greeks, and contains underlying price data for indices, as available to historical purchasers or subscribers with a Cboe Global Indices (CGI) license.
- underlying_options_eodcalcs_2023_08_25_no_cgi_subscription: 34 columns — End-of-day summary for option series, including OHLC, volume, open interest, and two market snapshots (15:45 ET and EOD). This version includes calculated values like Implied Volatility and Greeks, but, as per the `no_cgi_subscription` naming, it does not contain underlying price data for indices, which are reported as zero.
- underlying_trades: 9 columns — Tick-level trade data for an underlying security. Each row represents a single trade, providing price, size, exchange, and the prevailing bid/ask at the time of the trade.
- vix_daily_inputs: 19 columns — Daily input data used in the calculation of the Cboe Volatility Index (VIX). It includes the options series (SPXW) that were part of the calculation, their prices, and their individual contribution to the final index value.
- volatility_surfaces_constant_maturity_delta_relative: 49 columns — Intraday snapshots of the implied volatility surface for a specific underlying (SPY), structured by constant maturity and constant delta. Each row represents a point in time for a specific maturity (e.g., 1M), showing the implied volatility and strike for various delta levels (e.g., 5-delta, 10-delta).
- volatility_surfaces_constant_maturity_price_relative: 55 columns — Intraday snapshots of the implied volatility surface for a specific underlying (SPY), structured by constant maturity and strike price relative to the underlying price. Each row represents a point in time for a specific maturity (e.g., 1M), showing the implied volatility for various strike levels expressed as a percentage of the underlying price (e.g., 90% of spot, 110% of spot).
- volatility_surfaces_expiration_specific_delta_relative: 49 columns — Intraday snapshots of the implied volatility surface for a specific underlying (SPY), structured by specific expiration date and constant delta. This provides a raw, un-interpolated view of the volatility smile for each listed expiry.
- volatility_surfaces_expiration_specific_price_relative: 55 columns — Intraday snapshots of the implied volatility surface for a specific underlying (SPY), structured by specific expiration date and strike price relative to the underlying price. This provides a raw, un-interpolated view of the volatility smile for each listed expiry against relative strike levels.
- bbo_2020_01_02_cadjpy: 5 columns — Headerless Best Bid and Offer (BBO) tick data for the CAD/JPY currency pair for a specific trading session. Each row is a snapshot of the BBO.
- bbo_ldn_2023_08_22_cadjpy: 5 columns — Headerless Best Bid and Offer (BBO) tick data for the CAD/JPY currency pair from the London (ldn) trading session. Each row is a snapshot of the BBO.
- bbo_ny_2020_01_02_cadjpy: 5 columns — Headerless Best Bid and Offer (BBO) tick data for the CAD/JPY currency pair from the New York (ny) trading session. Each row is a snapshot of the BBO.
- cccy_eod: 6 columns — End-of-day (EOD) data for various cryptocurrency symbols. Provides daily open, high, low, and close prices.
- cccy_tick: 3 columns — Tick-level data for various cryptocurrency symbols. Each row represents a price update at a specific point in time.
- cgi_eod: 6 columns — End-of-day (EOD) data for Cboe Global Indices (CGI). Provides daily open, high, low, and close values for various Cboe proprietary indices.
- cgi_tick: 3 columns — Tick-level data for Cboe Global Indices (CGI). Each row represents a calculated index value at a specific point in time.
- eod_theos_cboe_prop_20251215_1600: 15 columns — End-of-day theoretical option prices and other calculated values for Cboe proprietary products. Provides a snapshot of model-based pricing at a specific time (16:00).
- ftse_eod: 6 columns — End-of-day (EOD) data for FTSE Russell indices. Provides daily open, high, low, and close values.
- ftse_tick: 3 columns — Tick-level data for FTSE Russell indices. Each row represents a calculated index value at a specific point in time.
- inav_eod: 6 columns — End-of-day (EOD) data for Indicative Net Asset Values (iNAVs) of ETFs. Provides daily open, high, low, and close values for the iNAV.
- inav_tick: 3 columns — Tick-level data for Indicative Net Asset Values (iNAVs) of ETFs. Each row represents a calculated iNAV at a specific point in time.
- livevol_exchange_ids: 4 columns — Reference table mapping numeric exchange IDs to their display names, full names, and the types of securities they trade. This is used to enrich trade data files that contain only an `exchange_id`.
- livevol_trade_condition_ids: 3 columns — Master reference table mapping numeric trade condition IDs to their names and applicable security types. Used to enrich trade data containing a `trade_condition_id`.
- main_eod: 7 columns — End-of-day (EOD) data for main Cboe indices. Provides daily open, high, low, and close values, and may include total option volume for related options.
- main_tick: 4 columns — Tick-level data for main Cboe indices (e.g., VIX). Each row represents a calculated index value at a specific point in time.
- msci_eod: 6 columns — End-of-day (EOD) data for MSCI indices. Provides daily open, high, low, and close values.
- msci_tick: 3 columns — Tick-level data for MSCI indices. Each row represents a calculated index value at a specific point in time.
- mstar_eod: 6 columns — End-of-day (EOD) data for Morningstar (mstar) indices. Provides daily open, high, low, and close values.
- mstar_tick: 3 columns — Tick-level data for Morningstar (mstar) indices. Each row represents a calculated index value at a specific point in time.
- rts13_public_trade_data_apa: 38 columns — Public trade data from an Approved Publication Arrangement (APA) under MiFID II/MiFIR regulations (likely RTS 13). It contains post-trade transparency reports for financial instruments.
- snp_2020_01_02_cadjpy: 4 columns — Headerless snapshot data for the CAD/JPY currency pair. 'snp' likely stands for snapshot, capturing a price and size at a point in time, possibly the last trade or a quote update.
- snp_ldn_2023_08_22_cadjpy: 4 columns — Headerless snapshot data for the CAD/JPY currency pair from the London (ldn) session. 'snp' likely stands for snapshot, capturing a price and size at a point in time.
- snp_ny_2020_01_02_cadjpy: 4 columns — Headerless snapshot data for the CAD/JPY currency pair from the New York (ny) session. 'snp' likely stands for snapshot, capturing a price and size at a point in time.
- tks_2020_01_02_cadjpy: 6 columns — Headerless tick data ('tks') for the CAD/JPY currency pair from a legacy format. Includes a sequence ID and status flag along with trade price and size.
- tks_ldn_2023_08_22_cadjpy: 6 columns — Headerless tick data ('tks') for the CAD/JPY currency pair from the London (ldn) session. Includes a sequence ID and status flag along with trade price and size.
- tks_ny_2020_01_02_cadjpy: 6 columns — Headerless tick data ('tks') for the CAD/JPY currency pair from the New York (ny) session. Includes a sequence ID and status flag along with trade price and size.
