This flowchart represents a comprehensive machine learning pipeline for time series forecasting, decomposition, and anomaly detection. It outlines a systematic process that transforms raw temporal data into structured forecasts, isolated time series components, and evaluation metrics. The diagram is logically structured to flow from left to right, transitioning from raw inputs through distinct preprocessing and modeling phases, culminating in aggregated forecasts and specific machine learning outputs.

The diagram is organized into three primary, dashed-line visual groupings: "Preprocessing," "Modeling," and "Decomposed Time Series." On the far left, the process begins with two initial data stores represented as cylinders: the "Input Time Series" and an optional "Input Custom Holiday" dataset. On the far right and bottom left, the pipeline concludes with several distinct final output data stores, each associated with specific Google BigQuery ML functions denoted in blue text. Throughout the diagram, rectangular nodes represent active processing or algorithmic steps, while cylindrical nodes represent data datasets or intermediate states. 

The data flow begins when the "Input Time Series" enters the "Preprocessing" block. Here, the data undergoes a strict linear sequence of cleaning operations. First, it passes through "Duplicated Timestamp Handling," followed by "Data Frequency Inference." The data then moves to "Irregular Timestamp Handling" before concluding the preprocessing phase at "Missing Data Interpolation." 

Once preprocessed, the cleaned time series data enters the "Modeling" block, which functions as an iterative decomposition loop. The preprocessed data, combined with data from the "Input Custom Holiday" store, first enters the "Holiday Adjustment" process. This step extracts a "Holiday Component," which is stored in the "Decomposed Time Series" block, and outputs a "De-holiday'ed Time Series" that feeds into the next step. This pattern repeats sequentially. The "De-holiday'ed Time Series" undergoes "Spikes & Dips Outlier Cleaning," isolating an "Outlier Component" and passing forward a "De-outlier'ed Time Series." Next, "Seasonal & Trend Decomposition (STL)" extracts "Multiple Seasonal Components" and yields a "De-seasoned Time Series." This is followed by a "Step Change Adjustment" process, which removes a "Step Change Component" and advances a "Step-change Adjusted Time Series." Finally, this highly refined series undergoes "Trend Modeling (distributed auto.ARIMA)." 

The final "Trend Modeling" step acts as a major divergence point for data outputs and key relationships. It generates a "Residual Time Series" and extracts a "Trend Component" into the decomposed grouping. Additionally, it outputs directly to a standalone data store on the bottom left labeled "Evaluation Metrics + Model Coefficients," which is associated with the `ML.ARIMA_EVALUATE` and `ML.ARIMA_COEFFICIENTS` functions. Meanwhile, the "Decomposed Time Series" block serves as a central repository for all the isolated signals generated during modeling. 

The final outputs of the system are generated by utilizing these decomposed components. A pivotal operational node labeled "Aggregation" (represented by a circle with an 'X') combines "Point Forecast" data from the "Holiday Component" and "Multiple Seasonal Components," along with "Point Forecast + Intervals" from the "Trend Component." This aggregation produces the "Forecasted Time Series + Intervals" data store, tied to the `ML.FORECAST` function. Furthermore, the collective data within the "Decomposed Time Series" grouping is used wholesale to populate two other final output stores: one for "Seasonal, Trend, Holiday, Outlier, Step Change Components" (associated with `ML.EXPLAIN_FORECAST`), and another containing the "Fitted History Series + Forecasted Series + Intervals" (associated with `ML.DETECT_ANOMALIES`).
